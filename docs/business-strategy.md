## 1. 서비스 포지셔닝

### 1.1 핵심 가치 제안 (Value Proposition)

**"팀원은 하루 10초로 컨디션을 공유하고, 팀장은 뇌우가 오기 전에 먼저 알아챈다."**

Clima는 팀원의 심리적 컨디션을 날씨 메타포로 시각화하여, 팀장이 번아웃·이탈 신호를 선제적으로 감지하고 대응할 수 있게 하는 팀 컨디션 관리 도구이다.

핵심 가치 3가지:
- **초저마찰 체크인**: 1-100점 슬라이더 하나로 10초 완료. 설문 피로도 제로.
- **직관적 시각화**: 숫자가 아닌 날씨(맑음/구름조금/흐림/비/뇌우)로 팀 분위기를 한눈에 파악.
- **선제적 개입**: Niko-Niko 캘린더와 주간 추이, Jira 업무량 교차 분석으로 위험 신호 자동 감지 및 팀장 알림.

### 1.2 타겟 고객 (ICP: Ideal Customer Profile)

**Primary ICP — 10~50인 규모 IT/스타트업 팀의 팀장(Engineering Manager, PO, Chapter Lead)**
- 팀원 5~15명을 직접 관리
- 리모트/하이브리드 근무 환경에서 팀원 컨디션 파악이 어려움
- Jira, Slack 등 디지털 도구를 이미 사용 중
- 심리적 안전(psychological safety)에 관심이 있으나 실행 도구가 없음

**Secondary ICP — People Operations / HR 팀**
- 조직 전반의 웰빙 지표를 정량적으로 추적하고 싶은 HR 담당자
- 기존 분기별 설문(eNPS 등)의 실시간 보완재를 찾는 조직

**초기 집중 시장**: 한국 IT 스타트업/중견 테크 기업 (시리즈 A~C, 50~300명 규모)

### 1.3 경쟁 서비스 대비 차별점

| 기준 | Officevibe | 15Five | Lattice | **Clima** |
|------|-----------|--------|---------|-----------|
| 체크인 소요 시간 | 5~10분 (설문) | 15분 (주간 리포트) | 10분+ (OKR+피드백) | **10초** |
| 입력 방식 | 5점 리커트 | 텍스트 중심 | 복합 폼 | **점수 1개 + 선택 코멘트** |

| 팀장 알림 | 수동 확인 | 매니저 브리핑 | 수동 확인 | **자동 위험 감지 + Teams/Slack 알림** |

**핵심 차별화 요약:**
1. 해외 도구들은 '성과 관리 플랫폼'에 웰빙을 끼워넣은 구조. Clima는 **팀 컨디션 하나에 올인**하여 극단적으로 단순함.
2. 한국어 네이티브 + 한국 공휴일/근무 문화 반영 (주말/공휴일 자동 스킵).
3. Jira 연동으로 **업무 과부하 ↔ 컨디션 하락** 상관관계를 데이터로 증명.

### 1.4 날씨 메타포의 고유 강점

날씨 메타포는 단순한 UI 장식이 아니라 Clima의 핵심 경쟁 우위이다:

1. **유니버설 언어**: "오늘 비 와요"는 문화권/직급을 초월해 즉시 이해된다. 숫자 52점보다 "흐림"이 감정적 맥락을 훨씬 빠르게 전달한다.
2. **낙인 효과 감소**: "나 우울해"라고 말하기 어렵지만 "오늘 날씨가 흐려요"는 자연스럽다. 감정 표현의 심리적 장벽을 낮춘다.
3. **비교 불가능성**: 점수 72 vs 68은 비교와 경쟁을 유발하지만, "구름조금" vs "흐림"은 상태 인지에 집중하게 만든다.
4. **패턴 인식 가속**: Niko-Niko 캘린더에서 일주일 내내 비 아이콘이 이어지면 숫자 나열보다 **위기 신호가 시각적으로 즉시 포착**된다.
5. **대화 유발**: "오늘 날씨가 어때?"는 일상 인사와 자연스럽게 연결된다. 팀 내 컨디션 대화의 시작점 역할.

4. **알림 필요성**: 위험 팀원 수동 확인 불편 → 자동 알림 Pro 기능 안내

## 3. Go-To-Market 전략

### 3.1 PLG(Product-Led Growth) 선택 — 하이브리드 접근

**1차 전략: PLG (Product-Led Growth)**
- 이유: 체크인 도구는 **써봐야 가치를 느끼는 경험재**. 데모 영상이나 세일즈 피칭보다 직접 체험이 전환율이 높다.
- Free 티어로 진입 장벽 제거 → 팀장이 직접 팀 생성 → 팀원 초대 → 가치 체험 → Pro 전환.

**2차 전략: SLG 보완 (Enterprise)**
- 50인+ 조직은 의사결정 구조가 복잡하므로 세일즈 터치 필요.
- HR 담당자 or 조직문화팀 대상 데모 세션 + 파일럿 제공.

### 3.2 팀장 → 팀원 전파 시나리오

**Top-down (팀장 기점) — Primary Path**
```
팀장이 랜딩 페이지 방문
  → "우리 팀도 써보기" CTA 클릭
  → Free 팀 생성 (1분)
  → 팀원 5명 초대 (Slack/이메일 링크)
  → 팀원은 매일 10초 체크인
  → 팀장이 대시보드에서 가치 확인
  → 2주 후 Pro 전환 제안
```

**Bottom-up (팀원 기점) — Secondary Path**
```
팀원이 "팀에 제안하기" 폼 작성
  → Clima가 제안 키트 생성 (왜 필요한지 + 데모 링크)
  → 팀원이 팀장에게 공유
  → 팀장이 파일럿 시작
```

이 Bottom-up 경로가 현재 `/request-access` 페이지의 "팀원 → 팀에 제안하기" 기능과 정확히 매칭됨.

### 3.3 온보딩 퍼널

```
[인지] 랜딩 페이지 / 콘텐츠 마케팅 / 커뮤니티 바이럴
   ↓
[관심] "대시보드 미리보기" — 샘플 데이터로 날씨 시각화 체험
   ↓
[체험] Free 팀 생성 — 이메일만으로 가입, 5명까지 무료
   ↓
[활성화] 3일 연속 체크인 달성 — "팀 날씨 리포트" 첫 수신
   ↓  ← Activation Moment: 팀장이 처음으로 팀 전체 날씨를 보는 순간
[습관화] 주간 리포트 수신 + Niko-Niko 패턴 확인
   ↓
[전환] 5명 초과 or Jira 연동 필요 시 → Pro 업그레이드
   ↓
[확산] 다른 팀 팀장에게 추천 → 조직 내 확산
```

**핵심 지표:**
- Activation Rate: 팀 생성 후 3일 내 3명 이상 체크인한 팀 비율
- Week 1 Retention: 첫 주 5일 중 3일 이상 체크인한 팀원 비율
- Time to Value: 팀 생성 → 첫 "의미 있는 패턴 발견"까지 걸리는 시간 (목표: 5영업일)

## 4. 랜딩 페이지 전략

### 4.1 히어로 메시지 제안 (3가지 안)

**안 A — 현재 메시지 발전형 (감성 + 질문)**
> 우리 팀의 오늘 날씨, 맑음일까요?
> 팀원은 하루 10초로 컨디션을 공유하고, 팀장은 뇌우가 몰아치기 전에 먼저 알아챌 수 있어요.

평가: 현재 사용 중. 날씨 메타포를 직관적으로 전달하며 호기심을 유발. 다만 "도구"보다 "감성"에 치우쳐 B2B 신뢰감이 약할 수 있음.

**안 B — 문제 직시형 (Pain → Solution)**
> 팀원이 지쳐가고 있다는 걸, 퇴사 면담에서 처음 알았나요?
> Clima는 매일 10초 체크인으로 팀 컨디션을 날씨로 시각화합니다. 뇌우가 오기 전에, 먼저 다가가세요.

평가: B2B 의사결정자(팀장)의 실제 pain point를 정면으로 건드림. 전환율이 높을 수 있으나, 다소 무거운 톤.

**안 C — 숫자 + 메타포 하이브리드**
> 하루 10초, 팀의 날씨를 읽는 법
> 5단계 날씨로 팀 컨디션을 추적하고, 위험 신호를 자동으로 감지합니다. 설문 없이, 숫자 하나로.

평가: 기능 중심. 명확하고 깔끔하지만 감성 호소력이 약함.

**추천: 안 B를 메인으로, 안 A를 서브 카피로 병행.**

### 4.2 CTA 전략

현재 CTA: "우리 팀도 써보기" (→ 접근 요청 폼)

**제안하는 CTA 이원화:**

| 위치 | Primary CTA | Secondary CTA |
|------|------------|--------------|
| 히어로 | **무료로 시작하기** (Free 팀 생성) | 데모 대시보드 보기 |
| How It Works 하단 | **1분 만에 팀 만들기** | 작동 방식 더 알아보기 |
| CTA 배너 | **지금 무료로 시작** | 팀 화면 먼저 보기 |

핵심 원칙:
- Primary CTA는 **즉시 행동**(가입/팀 생성)으로 연결. "써보기"보다 "시작하기"가 전환율이 높음.
- Secondary CTA는 아직 확신이 없는 방문자를 위한 **저관여 체험**(데모 대시보드).
- 현재 `/request-access` 폼은 **셀프서비스 가입이 준비되기 전까지의 과도기 장치**로 유지. 셀프서비스 출시 후에는 바로 가입 플로우로 전환.

### 4.3 사회적 증거 (Testimonial) 섹션 구성

현재 랜딩에는 사회적 증거가 없음. 아래 구성을 히어로와 CTA 배너 사이에 추가 권장:

```

2. 인용 카드 2~3개
   - "팀원이 3일 연속 비였을 때 바로 1:1을 잡았어요. 예전엔 퇴사 면담에서야 알았을 일이에요."
     — OOO, 플랫폼팀 팀장
   - "체크인이 10초니까 매일 해도 부담이 없어요. 설문은 한 달에 한 번도 귀찮았는데."
     — OOO, 백엔드 개발자
   - "Jira 티켓 30개 밀린 팀원이 뇌우 상태였어요. 업무 재분배하고 나서 3일 만에 맑음으로 바뀌었습니다."
     — OOO, PO

3. 숫자 지표 (가용 데이터 축적 후)
   - "평균 체크인 소요 시간: 8초"
   - "주간 체크인 지속률: 87%"
   - "위험 신호 선제 감지율: 3.2일 앞서 발견"
```

   - Niko-Niko 기법의 기원 + 디지털 시대에 맞는 진화 + Clima 적용 사례.
   - 타겟: 애자일 코치, 스크럼 마스터. SEO: "niko-niko calendar", "팀 건강 지표".

3. **"리모트 팀장의 최대 고민: '다들 괜찮은 거 맞아?'"**
   - 하이브리드/리모트 환경에서 팀원 상태 파악의 어려움 + 해결 프레임워크.
   - 타겟: 리모트 팀 리더. SEO: "리모트 팀 관리", "원격근무 팀 소통".

4. **"매주 설문 보내지 마세요. 매일 10초가 더 정확합니다."**
   - 분기별/주간 설문의 한계 (회상 편향, 설문 피로) vs 일일 마이크로 체크인의 데이터 정확도.
   - 타겟: HR/People Ops. SEO: "직원 만족도 조사 대안", "eNPS 보완".

5. **"업무량과 컨디션의 상관관계: Jira 데이터가 알려주는 것"**
   - 미완료 티켓 수와 컨디션 점수 간 상관 분석 (익명화된 실제 데이터 기반).
   - 타겟: EM, CTO, HR. SEO: "개발자 번아웃 데이터", "업무량 관리".

### 5.2 SEO 키워드 전략

**Primary 키워드 (검색량 + 전환 의도 높음):**

- 직원 번아웃 예방
- 팀 건강 지표
- 일일 체크인 도구

**Long-tail 키워드 (경쟁 낮음 + 구매 의도 높음):**

- "리모트 팀 컨디션 체크 방법"
- "niko-niko 캘린더 디지털"
- "Jira 업무량 번아웃 분석"

- "팀장 팀원 상태 파악 방법"

**콘텐츠 SEO 원칙:**
- 한국어 블로그 + 영문 SEO (글로벌 확장 대비)

- 매주 1편 발행, 3개월 후 50+ 인덱스 페이지 목표

### 5.3 팀 문화 / 심리 안전 관련 콘텐츠

1. **"심리적 안전(Psychological Safety)이란? Google이 발견한 최고 팀의 비밀"**
   - Amy Edmondson의 연구 + Google Project Aristotle 인용.
   - Clima가 심리적 안전의 "측정 도구"로 기능하는 방식.

2. **"1:1 미팅 전에 이것만 확인하세요 — 팀원의 이번 주 날씨"**
   - 1:1 미팅의 효과를 높이는 실전 팁 + Clima 데이터 활용법.

3. **"조용한 퇴사(Quiet Quitting)의 신호, 데이터로 잡는 법"**
   - 체크인 빈도 감소, 점수 하락 패턴 → 이탈 위험 조기 감지.

4. **"팀 문화 개선, 어디서부터? '측정'부터 시작하세요"**
   - "측정하지 않으면 관리할 수 없다" — 팀 컨디션 정량화의 가치.

5. **인포그래픽 시리즈: "이번 달의 팀 날씨 리포트"**
   - 월간 익명화된 트렌드 데이터 공개 (IT 업계 평균 컨디션 추이 등).
   - 바이럴 + 백링크 유도 목적.
